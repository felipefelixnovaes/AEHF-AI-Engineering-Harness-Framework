# AEHF Constitution

The following principles are non-negotiable.

1. **Clarity over ambiguity**  
   Every deliverable must make scope, assumptions, and outcomes explicit.

2. **Incremental evolution over rewrite**  
   Prefer small, reversible changes unless rewrite is explicitly justified.

3. **Stable memory over repeated context dumping**  
   `docs/` is durable memory; do not rely on transient chat context.

4. **Depth on demand over one-size-fits-all process**  
   Classify task depth and produce only required artifacts for that depth.

5. **Explicit assumptions over hallucinated certainty**  
   Unknowns must be surfaced, not silently guessed.

6. **No silent drift between docs and code**  
   Behavior, specs, and operational docs must stay aligned.

7. **Governance must be lightweight and actionable**  
   Use ADRs, changelog, release checks, and risk prompts pragmatically.

8. **Tool adapters may vary; method remains universal**  
   Context delivery differs by tool, but AEHF principles do not.
