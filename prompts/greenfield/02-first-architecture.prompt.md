# 02 - First Architecture

## Purpose

Create a pragmatic initial architecture that supports near-term delivery and evolution.

## When to use

After project bootstrap and before feature implementation.

## Required inputs

- Bootstrap brief
- Constraints and quality attributes
- Known integrations

## Instructions to the AI

1. Propose high-level components and boundaries.
2. Map data flow and integration points.
3. State trade-offs and architecture assumptions.
4. Recommend ADRs that should be created now.
5. Define what should be deferred.

## Expected outputs

- Architecture sketch
- Boundary definitions
- ADR candidates

## Completion criteria

Architecture is actionable for first feature delivery without over-design.
